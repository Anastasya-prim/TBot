# TBot — бот мебельного салона

Telegram-бот на **Node.js** и **Telegraf**: квиз заявки на расчёт, проверка статуса заказа (демо-данные), FAQ, уведомление менеджеру.

## Требования

- [Node.js](https://nodejs.org/) **18+**
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка и запуск

1. Склонируйте репозиторий или откройте папку проекта.

2. Установите зависимости:

   ```bash
   npm install
   ```

3. Создайте файл `.env` в корне проекта (можно скопировать пример):

   ```bash
   copy .env.example .env
   ```

   В `.env` укажите:

   - `BOT_TOKEN` — токен бота (обязательно).
   - `MANAGER_CHAT_ID` — необязательно: числовой ID чата или пользователя Telegram, куда слать уведомления о новых заявках. Свой ID можно узнать у [@userinfobot](https://t.me/userinfobot).

4. Запуск:

   ```bash
   npm start
   ```

   Остановка: **Ctrl+C** в терминале.

## Команды в Telegram

- `/start` или `/menu` — главное меню.
- `/status` — запрос статуса по телефону, **ИД заявки** (З-…) или номеру договора (Д-…).

## Демо-данные для статуса заказа

Реестр в `src/data/business.js` (`ORDERS_REGISTRY`). Можно ввести **телефон**, **ИД** или **договор**:

| Телефон | ИД заявки | Договор | Этап |
|---------|-----------|---------|------|
| +79991234567 | З-1001 | Д-2024-001 | изготовление |
| +79001112233 | З-1002 | Д-2024-077 | готов к доставке (спросит удобное время) |
| +79005556666 | З-1003 | Д-2023-412 | замер |

После отправки квиза новой заявке присваивается ИД вида **З-2000**, **З-2001**, … — по нему тоже можно проверить статус (до перезапуска бота).

## Структура

- `src/index.js` — точка входа, загрузка `.env`, запуск polling.
- `src/bot.js` — сценарии и обработчики Telegraf.
- `src/sessionStore.js` — сессии пользователей в памяти процесса.
- `src/data/business.js` — захардкоженные FAQ, мок-заказы, массивы заявок и логов в памяти.
- `src/utils/phone.js` — нормализация и проверка телефона.

После перезапуска бота все сессии и накопленные в памяти заявки/логи обнуляются.
