# TBot — бот мебельного салона

Telegram-бот на **Node.js** и **Telegraf**: квиз заявки на расчёт, проверка статуса заказа, FAQ, уведомление менеджеру. Данные заявок и лидов хранятся в **SQLite** или **PostgreSQL** (в т.ч. Supabase).

## Требования

- [Node.js](https://nodejs.org/) **18+**
- [Redis](https://redis.io/) — сессии пользователей и счётчик номеров заявок (`REDIS_URL`)
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка и запуск

1. Клонируйте репозиторий или откройте папку проекта.

2. Установите зависимости:

   ```bash
   npm install
   ```

3. Создайте `.env` в корне (шаблон — `.env.example`):

   ```bash
   copy .env.example .env
   ```

   На Linux/macOS: `cp .env.example .env`

   **Обязательно:**

   - `BOT_TOKEN` — токен бота.
   - `REDIS_URL` — например `redis://127.0.0.1:6379`.

   **Хранилище заявок:**

   - По умолчанию — **SQLite**, файл `./data/tbot.db` (можно задать `SQLITE_PATH`).
   - Для **PostgreSQL** (в т.ч. Supabase) укажите `DATABASE_URL`; тогда SQLite не используется. Подробности и варианты подключения — в комментариях `.env.example`.

   **По желанию:**

   - `MANAGER_CHAT_ID` — числовой chat_id для уведомлений о новых заявках ([@userinfobot](https://t.me/userinfobot)).

4. Запуск:

   ```bash
   npm start
   ```

   Остановка: **Ctrl+C**.

5. Экспорт таблиц в CSV (из активного хранилища):

   ```bash
   npm run export:csv
   ```

## Команды в Telegram

- `/start` или `/menu` — главное меню.
- `/status` — статус по телефону, **номеру заявки** (цифры) или номеру договора (Д-…).

## Демо-заказы в пустой БД

При первом запуске на **пустой** базе в таблицу `orders` добавляются примеры — ими можно проверить «Статус заказа»:

| Телефон | Номер заявки | Договор | Этап |
|---------|--------------|---------|------|
| +79991234567 | 1001 | Д-2024-001 | изготовление |
| +79001112233 | 1002 | Д-2024-077 | готов к доставке (спросит удобное время) |
| +79005556666 | 1003 | Д-2023-412 | замер |

Новые заявки из квиза получают **порядковый номер** (2000, 2001, …) — счётчик в **Redis**. Старые записи с префиксом **З-…** в базе по-прежнему находятся по цифрам или по прежнему виду.

## Структура проекта

- `src/index.js` — загрузка `.env`, Redis, БД, схема, `initBusiness`, запуск long polling.
- `src/bot.js` — сценарии Telegraf (квиз, статус, FAQ).
- `src/sessionStore.js` — сессии в **Redis** (`tbot:sess:*`, TTL 7 дней); прогресс квиза дублируется в таблицу **`quiz_sessions`** в БД.
- `src/data/business.js` — справочники и кэш заказов, выдача номера заявки, поиск заказа.
- `src/db/store.js` — выбор SQLite или PostgreSQL; реализации в `storeSqlite.js` / `storePg.js`.
- `src/utils/phone.js`, `src/utils/orderId.js` — телефон и нормализация номеров заявок.

Перезапуск процесса **не** стирает данные в Redis и в БД. В оперативной памяти процесса держится только краткоживущий кэш списка заказов в `business.js` (обновляется автоматически).
